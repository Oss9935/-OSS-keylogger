간단 내용 : 여기서 커널 레벨 키로거 구현 방법을
notifier를 활용한 방식, tty를 이용한 방식의 원리에 대해 설명함.

* 영어 논문.... : 그래도 시간있으면 꼭 읽어보는것이 도움될 듯!!

(참고할만한 내용)

>>> LKM은 특정 시스템 이벤트가 발생할 때
커널의 다른 부분으로부터 알림을 요청할 수 있습다
이 메커니즘은 초기에 Linux v.1.1에서 개발되어
관련된 커널 모듈에 네트워크 이벤트에 대한 정보를 제공한다.

>>> notifier chain은 콜백함수로 이루어진 링크드 리스트

>>> 커널은 리스트의 헤드 유지!

>>> 이벤트 발생 -> 탐지 모듈이 리스트 탐색하여 각 기능 호출

>>> 모듈을 notifier chain에 등록 하는 방법 : 콜백함수의 포인터를 엔트리에 추가
=> keylogger 코드에서 notifier_block 구조체의 '.notifier_call' 에 등록하는 부분!!

>>> 필요 없는은 모듈 notifier chain에서 콜백함수 제거
